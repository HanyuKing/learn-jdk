## 四个核心目标
1. 实物奖数量固定，要尽量应发尽发。
2. 活动有时间范围，前期不能把实物奖发完，后期也要有非兜底奖。
3. 保底策略必须启用：任何用户连续 x 次未中实物奖时，下一次必触发保底命中（100% 保障）。
4. 单依纯演唱会门票：同一用户只能中一次，且支持指定 UID。

## 规则设计（围绕核心目标）

### 1) 兜底不设总数
- 兜底只是“剩余概率”的承载项，不需要总数。
- 实物奖发不出去时，概率自动回落给兜底。

### 2) 时间释放（确保前后期都有奖）
把活动时间分成三段，控制可释放量：
- 前期（0%~60%）：最多放出 50% 实物奖
- 中期（60%~85%）：再放出 30%
- 末期（85%~100%）：放出剩余 20%

规则：
- 每个奖品的“可用数量” = 当前阶段允许释放的数量 - 已发数量
- 可用数量为 0 时，该奖品当前不可中

### 3) 权重抽取（基于截图权重）
- 有可用数量的奖品才参与抽取
- 按截图概率做权重分配
- 兜底 = 1 - 其他奖品概率之和

### 4) 阶段追差（尽量应发尽发）
- 阶段结束时，若某奖品未达到该阶段配额，则差额顺延到下一阶段
- 通过阶段追差提升发放完成度，但不做“末期强制提高概率”
- 若活动结束仍有库存，可结转到下一期或用于后续活动

### 5) 连续未中保底（必须启用，100% 生效）
目标：降低用户“连续多次兜底”的挫败感，但不破坏整体库存节奏。  
做法：当用户连续 x 次未中实物奖时，下一次必触发“保底命中”，但仅限“可用量 > 0”的非兜底奖品（通常限定为小奖/中等奖）。  

建议逻辑：
- 仅对“非兜底奖”生效，且可配置排除大奖，避免稀缺奖被保底消耗。
- 保底命中依然遵守“阶段可用量”，若可用量为 0，则保底延后。
- 保底命中优先使用小奖，若小奖不可用再升到中等奖。

“x” 的取值说明（满足 100% 保底）：  
- 目标“100% 用户在 x+1 次内至少中 1 次”由保底规则保证，x 是硬阈值。  
- x 的大小影响保底触发频率与成本，建议用“自然未中概率”评估成本：  
  r = (1 - p)^x，其中 p 为非兜底中奖率（截图 p = 8.52%）。  
- 示例（用于评估成本，不是硬规则）：  
  - x = 19 ⇒ r ≈ 20%（高体验、高成本）  
  - x = 26 ⇒ r ≈ 10%（平衡档）  
  - x = 34 ⇒ r ≈ 5%（低成本）  
- 若 r = 100%，表示“不限制保底触发比例”，x 需由产品/成本直接指定。

### 6) 单依纯演唱会门票用户限制与指定 UID
- 同一用户最多中 1 次该奖品。
- 若配置“指定 UID 列表”，则名单内用户首次抽奖 100% 命中该奖品（仍需有库存与可用量）。  
  该用户命中过一次后，再次抽奖走正常抽奖逻辑，并自动排除该奖品权重。  
- 名单外用户不能抽中该奖品，该奖品权重对其直接排除。  
- 指定 UID 仅影响该奖品的中奖规则，不改变其他奖品的抽取逻辑。

## 抽奖逻辑举例（覆盖场景的完整示例）
以下示例均使用基准权重总和 13000，随机数取 1~13000 的整数。  

### 场景 A：前期配额已用尽（只有兜底）
时间进度 20%，前期配额已被快速抽完，所有实物奖“可用量=0”。  
随机数区间只剩：1~13000 为兜底。  
示例抽取：随机数 8888，结果为“想要票数*1”。  

### 场景 B：前期正常抽中小奖
时间进度 30%，前期配额仍有可用量，所有奖品都可抽。  
随机数区间如下：  
1~2：单依纯演唱会门票*2  
3~4：游乐场门票*2  
5~7：超大公仔（一对）  
8~47：毛绒挂件  
48~57：早安机  
58~82：毛绒背包  
83~107：保温杯  
108~1107：亚克力相框挂件  
1108~13000：想要票数*1  
示例抽取：随机数 600，命中“小奖-亚克力相框挂件”，库存扣减成功即发放。  

### 场景 C：中期追差（阶段差额顺延）
时间进度 70%，前期小奖配额 500 已发 430，差额 70 顺延到中期。  
中期小奖可用量 = 300 + 70 - 已发（中期）120 = 250。  
同时假设“早安机”已发完，权重移除，总权重变为 12990。  
新的随机数区间如下：  
1~2：单依纯演唱会门票*2  
3~4：游乐场门票*2  
5~7：超大公仔（一对）  
8~47：毛绒挂件  
48~72：毛绒背包  
73~97：保温杯  
98~1097：亚克力相框挂件  
1098~12990：想要票数*1  
示例抽取：随机数 60，命中“毛绒背包”，库存扣减成功即发放。  

### 场景 D：末期仍有奖品，但不做末期强制提升
时间进度 92%，大奖已全部发完，中等奖仅剩少量，小奖仍有库存。  
此时仍按权重分配，不做“末期强制提高概率”，兜底仍占大头。  
示例抽取：随机数 12000，落在兜底区间，结果为“想要票数*1”。  

### 场景 E：命中但并发抢光（回落兜底）
时间进度 80%，抽中“小奖-亚克力相框挂件”，但此刻库存被并发抢光。  
库存扣减失败，结果回落为“想要票数*1”。  

### 场景 F：连续未中触发保底
用户已连续 x 次未中实物奖（达到保底阈值）。  
下一次抽取必进入“保底命中流程”，优先发放小奖；若小奖可用量为 0，则尝试中等奖。  
若所有非兜底奖均不可用，则保底延后，等待库存可用。  

### 场景 G：单依纯演唱会门票的 UID 限制
用户 A 在指定 UID 名单内，第一次抽奖直接命中“单依纯演唱会门票*2”（需有库存与可用量）。  
用户 A 命中过一次后，第二次抽奖回到正常逻辑，该奖品权重自动排除。  
用户 B 不在指定 UID 名单内，该奖品对其始终不可中。  

### 场景 H：指定 UID 但库存不足
用户 C 在指定 UID 名单内，首次抽奖应 100% 命中该奖品。  
但此时该奖品库存不足或可用量为 0，则“指定 UID 直中”不触发，回退到正常抽奖逻辑。  
后续若库存恢复且用户 C 仍未中过该奖品，则下一次抽奖仍可触发 100% 命中。  

## 可配置参数（最少化）
- 三段时间比例（建议 60% / 25% / 15%）
- 各奖品阶段配额比例（建议 50% / 30% / 20%）
- 保底阈值 x（建议按体验与成本选择）
- 单依纯演唱会门票指定 UID 列表

## 运营配置建议（按成本档位给 x / r 建议值）
说明：r 表示“自然未中概率”，用来评估保底触发成本。  
参考 p = 8.52%（截图非兜底概率），r = (1 - p)^x。  

低成本档（更省预算）：  
- x ≈ 34 ⇒ r ≈ 5%  
- 适用于预算紧、奖品价值较高的活动  

中成本档（平衡体验与成本）：  
- x ≈ 26 ⇒ r ≈ 10%  
- 适用于常规营销活动  

高成本档（强体验保障）：  
- x ≈ 19 ⇒ r ≈ 20%  
- 适用于高留存或强品牌活动  

不限制档（体验优先）：  
- r = 100%，x 需人工指定（例如 10~20）  
- 适用于追求“强保底”体验的活动
